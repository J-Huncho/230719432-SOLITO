Next.js App Router Documentation

Welcome to the Next.js App Router documentation. This guide will help you understand and implement the latest routing system in Next.js applications.

Getting Started

To start using the App Router, create a new Next.js project or update an existing one to the latest version. Then, create an `app` directory in your project root.

                
                  {`npx create-next-app@latest my-app --use-npm
cd my-app
npm run dev`}

Routing

The App Router uses a file-system based routing mechanism. Each folder in the `app` directory represents a route segment, and the hierarchy defines the URL structure.

  • Create folders for each route segment
  • Use `page.js` files to define the UI for a route
  • Nested routes are created by nesting folders

Pages and Layouts

Pages are UI unique to a route, while layouts are shared UI between multiple pages.

  • Create a `page.js` file in a route folder to define a page
  • Use `layout.js` files to create shared layouts
  • Layouts can be nested for more complex UI structures

Data Fetching

Next.js provides powerful data fetching mechanisms that work seamlessly with the App Router.

  • Use `fetch()` for client-side and server-side data fetching
  • Implement React Server Components for efficient server-side rendering
  • Utilize `useEffect` for client-side only data fetching

Rendering

The App Router supports both server-side and client-side rendering strategies.

  • Server Components are the default for improved performance
  • Use the `'use client'` directive for Client Components
  • Mix Server and Client Components for optimal user experiences

Caching

Next.js provides built-in caching mechanisms to improve performance and reduce server load.

  • Route Cache for faster navigation between pages
  • Data Cache for storing fetch requests
  • Full Route Cache for static routes
